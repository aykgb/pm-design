PM System Kit Architecture

OpenCode + Worktree Pool + Multi-Agent Async Collaboration · Self-Healing Runtime

Core Runtime

  • • OpenCode Server — session lifecycle, agent dispatch, prompt_async API
  • • session-worktree-mgr — CLI hub: pool prepare/dispatch/release, overview
  • • Session Sidecar — status tracker: idle/busy/streaming, notify PM
  • • Worktree Pool — 10 isolated worktrees with independent git branches

Self-Healing

  • • Stuck Detection — session busy > 15min with no time.updated → notify PM
  • • Auto Recovery — --force hard-delete stuck session + recreate fresh
  • • Idle Watch — auto idle-watch spawned on dispatch, notify on busy→idle
  • • Stale Cleanup — tombstoned sessions pruned, >1d sessions archived on release

PM Agent Injection

  • • persona.md — management & casual chat modes, tone, boundaries
  • • operational_conventions.md — 5 OC groups (沟通/分支/派发/模式/规范)
  • • project_memory.md — interaction history, agent dispatch templates, decisions
  • • user_profile.md · user_behavior.md — cross-session memory persistence
