PM System Architecture

OpenCode + Worktree Pool + Multi-Agent Async · .pm/design v4 · 5 planes · 3 paths · 1 self-heal loop

派发路径 (PM 主动)

  • • CLI: pool prepare --branch feat_xxx
  • • dispatch: pool dispatch wt_N Daedalus --task "..." --yes
  • • HTTP: POST /session/{sid}/prompt_async
  • • agent 在独立 wt 异步工作，PM 不等结果

通知路径 (PM 自动)

  • • watch_session() 6 调用点统一入口
  • • sidecar 5s 轮询 OpenCode 状态
  • • busy→idle 跳变 触发 idle-watch 进程
  • • PM 收到通知后自动 dispatch 下一阶段

PR 路径 (外部)

  • • 1 PR = 1 Batch · gh pr create
  • • Step 3.5: Codex bot 自动 review
  • • check-codex.sh 拉 comments
  • • merge: 开发者执行 · PM 禁 merge (OC2.5)

自愈路径 (兜底)

  • • stuck 检测: busy > 15min 自动 hard-delete
  • • --force: 跳过 staleness 检查直接重建
  • • pm-guardian.js: OpenCode plugin 守护 + 日志
  • • recreate_always: prepare 时无条件重建
